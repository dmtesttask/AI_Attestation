# Автоматичне розгортання OpenClaw на GCP 🚀

Цей каталог містить скрипти для автоматичного створення безкоштовної віртуальної машини в Google Cloud Platform, встановлення Node.js, налаштування OpenClaw та автоматичного завантаження всіх необхідних агентів та правил оркестрації.

## 📋 Системні вимоги та підготовка

Перед запуском скрипта переконайтеся, що у вас є:
1. **Акаунт Google Cloud Platform** з прив'язаною платіжною карткою (гроші списуватися не будуть, e2-micro в us-central1 повністю безкоштовний).
2. **API-ключ Gemini** (отримайте на [Google AI Studio](https://aistudio.google.com/)).
3. **Токен Telegram-бота** (створіть через `@BotFather`).

---

## ⚡ Інструкція із запуску

1. Увійдіть у **[GCP Console](https://console.cloud.google.com/)**.
2. У правому верхньому кутку натисніть іконку **Activate Cloud Shell** (>_), щоб відкрити термінал у браузері.
3. Зробіть клон вашого репозиторію з проєктом:
   ```bash
   git clone https://github.com/dmtesttask/AI_Attestation.git OpenClaw
   ```
4. Перейдіть до каталогу авторозгортання:
   ```bash
   cd OpenClaw
   ```
5. Зробіть скрипт виконуваним та запустіть його:
   ```bash
   chmod +x setup-gcp.sh
   ./setup-gcp.sh
   ```

---

## ⚙️ Що робить скрипт?

1. **Вмикає API**: Активує служби Compute Engine та Secret Manager у вашому проєкті GCP.
2. **Налаштовує Secret Manager**: Безпечно зберігає API-ключ Gemini та токен Telegram-бота (запитуються в консолі при першому запуску). Ключі ніколи не потраплять у вихідний код!
3. **Створює права доступу**: Налаштовує спеціальний сервісний акаунт ВМ із мінімальними правами (доступ лише до цих двох секретів).
4. **Створює ВМ**: Запускає віртуальну машину `e2-micro` (us-central1-a, Ubuntu 24.04, 30 GB Standard HDD) під безкоштовний тариф GCP Always Free.
5. **Ініціалізує сервер (ВМ)**:
   - Оновлює пакети, встановлює Node.js 22 та Git.
   - Встановлює OpenClaw CLI глобально.
   - Створює ізольованого системного користувача `openclaw` для запуску сервісу.
   - Отримує ключі зі Secret Manager та проводить неінтерактивну ініціалізацію (`openclaw onboard`).
   - Прописує в автозапуск `systemd` сервіс `openclaw.service`.
   - Встановлює глобальний аліас `openclaw` для всіх користувачів ВМ, щоб команди CLI автоматично виконувалися від імені потрібного користувача.
6. **Завантажує агентів**: Синхронізує файли персонажів критиків з папки `config/` на віртуальну машину.

## 🔗 Підключення до Telegram (Pairing)

Після успішного виконання скрипта:
1. Знайдіть вашого бота в Telegram та надішліть йому будь-яке повідомлення (наприклад, `Привіт`).
2. Бот відповість вам повідомленням з **кодом сопряження** (Pairing Code).
3. Підключіться до віртуальної машини через Cloud Shell:
   ```bash
   gcloud compute ssh openclaw-server --zone=us-central1-a
   ```
4. Активуйте сопряження в терміналі ВМ (завдяки налаштованим аліасам команда виконається коректно):
   ```bash
   openclaw pairing approve telegram <ВАШ_КОД>
   ```

## 🛠️ Управління та діагностика на сервері

Після підключення через SSH вам доступні такі команди:
* **Перегляд логів Gateway** у реальному часі (найзручніший спосіб бачити діалоги та запити):
  ```bash
  journalctl -u openclaw -f
  ```
* **Перезапуск служби** (наприклад, після ручної зміни конфігурації):
  ```bash
  sudo systemctl restart openclaw
  ```
* **Діагностика системи**:
  ```bash
  openclaw doctor
  ```
* **Список активних агентів**:
  ```bash
  openclaw agents list
  ```
