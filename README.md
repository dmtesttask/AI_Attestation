# Автоматичне розгортання OpenClaw на GCP 🚀

Цей каталог містить скрипти для автоматичного створення безкоштовної віртуальної машини в Google Cloud Platform, встановлення Node.js, налаштування OpenClaw та автоматичного завантаження всіх необхідних агентів та правил оркестрації.

## 📋 Системні вимоги та підготовка

Перед запуском скрипта переконайтеся, що у вас є:
1. **Акаунт Google Cloud Platform** з прив'язаною платіжною карткою.
2. **API-ключ Gemini** (отримайте на [Google AI Studio](https://aistudio.google.com/)).
3. **Токен Telegram-бота** (створіть через `@BotFather`).

---

## ⚡ Інструкція із запуску

1. Увійдіть у **[GCP Console](https://console.cloud.google.com/)**.
2. У правому верхньому кутку натисніть іконку **Activate Cloud Shell** (>_), щоб відкрити термінал у браузері.
3. Зробіть клон репозиторію з проєктом:
   ```bash
   git clone https://github.com/dmtesttask/AI_Attestation.git OpenClaw
   ```
4. Перейдіть до каталогу авторозгортання:
   ```bash
   cd OpenClaw
   ```
5. Зробіть скрипт виконуваним та запустіть його:
   ```bash
   chmod +x setup-gcp.sh
   ./setup-gcp.sh
   ```

---

## ⚙️ Що робить скрипт?

1. **Вмикає API**: Активує служби Compute Engine та Secret Manager у вашому проєкті GCP.
2. **Налаштовує Secret Manager**: Безпечно зберігає API-ключ Gemini та токен Telegram-бота (запитуються в консолі при першому запуску). Ключі ніколи не потраплять у вихідний код!
3. **Створює права доступу**: Налаштовує спеціальний сервісний акаунт ВМ із мінімальними правами (доступ лише до цих двох секретів).
4. **Створює ВМ**: Запускає віртуальну машину (us-central1, Ubuntu 24.04, 30 GB Standard HDD).
5. **Ініціалізує сервер (ВМ)**:
   - Оновлює пакети, встановлює Node.js 24 та Git.
   - Встановлює OpenClaw CLI глобально.
   - Створює ізольованого системного користувача `openclaw` для запуску сервісу.
   - Отримує ключі зі Secret Manager та проводить неінтерактивну ініціалізацію (`openclaw onboard`).
   - Прописує в автозапуск `systemd` сервіс `openclaw.service`.
   - Встановлює глобальний аліас `openclaw` для всіх користувачів ВМ, щоб команди CLI автоматично виконувалися від імені потрібного користувача.
6. **Завантажує агентів**: Синхронізує файли персонажів критиків з папки `config/` на віртуальну машину.

## 🔗 Підключення до Telegram (Pairing)

Після успішного виконання скрипта:
1. Знайдіть вашого бота в Telegram та надішліть йому будь-яке повідомлення (наприклад, `Привіт`).
2. Бот відповість вам повідомленням з **кодом** (Pairing Code).
3. Підключіться до віртуальної машини через Cloud Shell:
   ```bash
   gcloud compute ssh openclaw-server --zone=us-central1-a
   ```
4. Активуйте з'єднання бота в терміналі ВМ (завдяки налаштованим аліасам команда виконається коректно):
   ```bash
   openclaw pairing approve telegram <ВАШ_КОД>
   ```

## 👥 Налаштування ролей та оркестрації

OpenClaw використовує гнучку систему налаштування персонажів (ролей) та сценаріїв їхньої взаємодії за допомогою звичайних Markdown-файлів.

### 1. Створення та конфігурація ролей (Агентів)
Усі ролі членів комісії або критиків знаходяться в каталозі `config/agents/`. Кожна роль представлена окремою папкою (наприклад, `thesis-practitioner`), яка **обов'язково** містить два файли:
*   **`IDENTITY.md` (Хто це?):** Визначає ім'я, смайл-аватар, позиціонування та сферу відповідальності агента.
*   **`SOUL.md` (Характер та поведінка):** Описує стиль спілкування (наприклад, офіційний чи прискіпливий), тон та обмеження (наприклад, задавати лише одне запитання за раз).

### 2. Сценарій взаємодії (Оркестрація)
Логіка того, як саме бот реагує на команди користувача, прописана у файлі `config/workspace/AGENTS.md`. У ньому зазначено:
*   Які команди доступні студенту (наприклад, `/defend` для початку захисту, `/end` для завершення).
*   Послідовність виклику агентів за допомогою системних інструментів `sessions_spawn` (запустити агента) та `sessions_yield` (чекати на відповідь).
*   Формування фінального звіту модератором (`@session-moderator`) при завершенні захисту.

> [!IMPORTANT]
> **Дозвіл на запуск суб-агентів:**
> OpenClaw не дозволить запустити агента, якщо його немає у списку дозволених на сервері. Цей список налаштовується в інсталяційному файлі `vm-startup.sh` (параметр `agents.defaults.subagents.allowAgents`). Якщо ви додаєте або видаляєте агента, його ім'я необхідно оновити в цьому списку.

### 3. Як додати або видалити роль
*   **Щоб додати нову роль (наприклад, `music-composer`):**
    1. Створіть папку `config/agents/music-composer/` з файлами `IDENTITY.md` та `SOUL.md`.
    2. Додайте виклик `@music-composer` у потрібне місце сценарію в `config/workspace/AGENTS.md`.
    3. Додайте `"music-composer"` до списку `allowAgents` в конфігурації сервера (або у файлі `vm-startup.sh`).
*   **Щоб видалити роль:**
    1. Видаліть її виклик із файлу `config/workspace/AGENTS.md`.
    2. Видаліть папку ролі з `config/agents/` (за бажанням).
    3. Приберіть її назву зі списку `allowAgents`.

## 🔄 Багатокористувацький режим та аналіз матеріалів

За замовчуванням система налаштована на послідовну роботу з одним користувачем та потребує ручного підтвердження для кожного нового чату. Ви можете адаптувати її під свої потреби.

### 1. Захист для кількох студентів паралельно
Якщо ви хочете, щоб декілька студентів могли одночасно проходити захист у приватних повідомленнях з ботом без переплутування контексту та без ручного введення кодів сопряження (`pairing`), підключіться до ВМ через SSH та виконайте:

```bash
# 1. Розділити сесії для кожного користувача окремо
openclaw config set session.dmScope "per-channel-peer"

# 2. Дозволити всім користувачам писати боту без підтвердження коду сопряження
openclaw config set channels.telegram.dmPolicy "open"

# 3. Перезапустити сервіс OpenClaw для застосування змін
sudo systemctl restart openclaw
```

> [!WARNING]
> Режим `dmPolicy "open"` дозволяє будь-кому в Telegram взаємодіяти з вашим ботом, що може призвести до швидкого витрачання лімітів вашого Gemini API.

### 2. Надсилання додаткових матеріалів (PDF, презентації)
Завдяки мультимодальності моделі **Gemini 3.1 Flash Lite**, віртуальна комісія може аналізувати не лише текстові повідомлення, а й додані файли:
*   **Текст доповіді/роботи:** Можна скопіювати та надіслати безпосередньо в чат після команди `/defend` (контекст моделі дозволяє обробляти сотні сторінок тексту за раз).
*   **Файли роботи та слайди:** Студент може надіслати у чат файл роботи в форматі **PDF** або зображення слайдів презентації. Бот автоматично передасть їх комісії для детального аналізу та формування точкових запитань.

## 🛠️ Управління та діагностика на сервері

Після підключення через SSH вам доступні такі команди:
* **Перегляд логів Gateway** у реальному часі (найзручніший спосіб бачити діалоги та запити):
  ```bash
  journalctl -u openclaw -f
  ```
* **Перезапуск служби** (наприклад, після ручної зміни конфігурації):
  ```bash
  sudo systemctl restart openclaw
  ```
* **Діагностика системи**:
  ```bash
  openclaw doctor
  ```
* **Список активних агентів**:
  ```bash
  openclaw agents list
  ```
