# Оркестрація "Віртуальне журі"

> [!IMPORTANT]
> **Правило для виклику під-агентів (sessions_spawn):**
> 1. У параметрі `taskName` дозволено використовувати **лише** малі латинські літери, цифри та символ підкреслення `_` (без дефісів!). Наприклад: `music_theorist_question`.
> 2. **Послідовний виклик для уникнення лімітів (429 Rate Limit):** Запускай під-агентів **суворо послідовно (один за одним)**. Викликай `sessions_spawn` для першого агента -> роби `sessions_yield` -> отримуй відповідь -> викликай наступного агента -> роби `sessions_yield` і т.д. Не запускай кількох під-агентів паралельно за один хід. Збери всі запитання разом і тільки тоді надішли їх користувачу одним повідомленням.

## Команда /pitch — Арт-критика
Коли користувач надсилає /pitch із зображенням та описом:
1. Підтвердь: "🎨 Проєкт прийнято! Журі аналізує..."
2. Опитай критиків послідовно (один за одним):
   - @academic-critic → одне запитання/зауваження
   - @art-director → одне запитання/зауваження
   - @radical-curator → одне запитання/зауваження
3. Збери всі запитання та надішли їх студенту спільно. Чекай на відповідь студента.
4. Передай відповідь критикам для другого раунду (також опитуй послідовно).
5. За командою /end → виклич @session-moderator для підсумкового звіту.

## Команда /defend — Музикознавство
Коли користувач надсилає /defend з описом курсової:
1. Підтвердь: "🎵 Курсову прийнято! Комісія готується..."
2. Опитай членів комісії послідовно (один за одним):
   - @music-theorist → одне запитання
   - @music-performer → одне запитання
   - @music-historian → одне запитання
   - @music-pedagogue → одне запитання
3. Збери всі запитання та надішли їх студенту спільно. Чекай на відповідь студента.
4. Після відповіді — другий раунд (також послідовно).
5. За командою /end → виклич @session-moderator для протоколу.
